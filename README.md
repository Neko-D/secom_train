## 1 项目背景
在晶圆制造流程中，从前端光刻、刻蚀、沉积到后端封装测试，每一道工序都会由数百个传感器实时记录温度、电压、电流、气体流量、压力、膜厚等工艺参数。晶圆在完成整线制造后进入晶圆测试及成品测试环节，由测试设备给出"通过（Pass）"或"失效（Fail）"的结果。
由于半导体制造的高良率特征，失效样本极低，呈现典型的 **极端不平衡二分类问题**。尽早识别出潜在失效晶圆，能够：

- 降低不必要的后续封装与测试成本；
- 及时追溯造成失效的工艺环节，为工程改善提供依据；
- 建立工艺参数与良率之间的量化模型；
- 逐步实现"虚拟量测"与"预测型维护"。

本项目使用的 **SECOM** 数据集是公开的半导体制造传感器数据基准数据集，广泛用于学术研究与工业项目验证。

## 4. 数据切分原则

解析 `secom_labels.data` 中的时间戳，并按真实时间顺序排序，再执行 70%/10%/20% 的训练、验证、测试切分。

| 数据集 | 样本数 | 失效样本数 | 时间范围 |
| --- | ---: | ---: | --- |
| 训练集 | 1096 | 78 | 2008-07-19 11:55:00 至 2008-09-26 02:26:00 |
| 验证集 | 157 | 9 | 2008-09-26 03:12:00 至 2008-10-02 19:25:00 |
| 测试集 | 314 | 17 | 2008-10-02 20:54:00 至 2008-10-17 06:07:00 |

该设计把测试集放在最后 20% 的未来时间窗口中，评估更接近真实投产后的时间外推表现。

## 5. 防止数据泄漏的约束

所有可学习的数据处理步骤均只在训练集上 `fit`，验证集和测试集只执行 `transform`。

```text
原始数据
  -> 时间排序
  -> 70% 训练 / 10% 验证 / 20% 测试
  -> 训练集 fit 缺失率过滤
  -> 训练集 fit KNNImputer(k=5)
  -> 添加行级统计特征
  -> 训练集 fit 方差过滤、相关性过滤、标准化
  -> 训练集 fit ANOVA 特征选择
  -> 多模型、多采样、多种子搜索
  -> 验证集选择阈值和最终方案
  -> 只训练并保存最终 5-seed ensemble 模型
  -> 测试集一次性最终评估
  -> 复用最终模型计算 ensemble SHAP 与 CV 分析
```

验证集只用于模型选择、阈值选择和策略比较；测试集不参与任何调参。

## 6. 预处理方案

| 步骤 | 说明 |
| --- | --- | --- |
| 缺失率过滤 | 仅根据训练集缺失率保留特征 |
| KNN插补 | 仅在训练集拟合，验证和测试只转换 |

## 7. 特征选择计划

当前主流程使用 ANOVA F-score 进行特征选择：

```text
FEATURE_SELECTION_MODE = "anova_f"
N_FEATURES = 200
```

选择原因是多轮严格时间切分实验和特征数消融显示，Top-200 在 Recall、Specificity 与 G-Mean 之间更均衡。过少特征会明显压低 Recall；直接使用全部 312 个预处理特征会提高误报，G-Mean 反而下降。

最新训练集互信息 Top-5 特征为：

```text
['41', '570', '40', '308', '127']
```

这些互信息特征用于辅助理解数据结构，最终入模特征仍以 ANOVA F-score Top-200 为准。

## 9. 阈值与最终模型选择

验证集只有 9 个失效样本，因此阈值约束不能过硬。预定为0.67：

```text
VALIDATION_RECALL_FLOOR = 0.67
```

在 9 个验证集失效样本下，0.67 或 0.70 都对应至少检出 7 个失效样本；如果设为 0.78，则必须至少检出 8 个失效样本，阈值会更激进，通常会带来更多误报。

阈值搜索逻辑为：

1. 先筛选满足验证集 Recall 下限的阈值；
2. 在满足 Recall 下限的候选中优先最大化 G-Mean；
3. G-Mean 接近时参考 Precision 与 F1；
4. 若没有阈值满足 Recall 下限，则回退到最大化 G-Mean。

进行Stacking 与简单平均集成的比较，并与最优单模型进行对比。

最终阶段：只训练并保存最优模型的 5 个 seed 模型。

## 11. 最终评估方案

- 混淆矩阵：TN / FP / FN / TP
- 核心指标：Recall、Precision、G-Mean、F1、Specificity、AUC-ROC、AUC-PR、MCC、Balanced Accuracy、Brier Loss
- 可视化：ROC 曲线、PR 曲线、混淆矩阵热力图、校准曲线

结果适合作为高召回初筛模型，先尽可能拦截失效风险样本，再交由复检、工艺工程或更高成本检测环节进一步确认。

## 12. SHAP 可解释性与关键特征挖掘

- 基于最终模型计算测试集所有样本的 SHAP 值；
- 输出 Top-20 全局 SHAP 重要性（关键失效特征）；
- 绘制 SHAP Beeswarm 图，展示每个特征取值高低对"失效"预测概率的贡献方向与强度。
- CV 与 SHAP 联合分析用于识别既重要又波动大的传感器。
这些特征应优先映射到真实工艺参数、机台、腔体、批次和维护记录中，作为后续工艺改善入口。

## 14. 后续优化方向

1. 降低模型负担：根据结果可优先保留top级中出现过的配置，未出现配置可列为备用配置。
2. 提升 Precision：当前仅靠传感器数值特征较难突破，更需要引入机台、腔体、批次、产品类型、工序段、维护记录等上下文。
3. 工艺映射：将 F59、F205、F486、F247、F460 等特征映射到真实传感器含义，判断是否存在可操作的工艺干预点。

## 15. 结论

本项目围绕 SECOM 数据集，构建了一条从时间外推划分 + 训练集内防泄漏预处理 + 特征选择 + 最佳模型确认 + SHAP/CV 传感器分析的完整机器学习流水线。核心目标是在保证recall ＞90%的情况下，最大化综合分类表现（AUC-ROC ≥ 0.70，G-Mean ≥ 0.70），并为工艺工程师提供可解释的关键失效特征列表。
