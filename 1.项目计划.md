# 半导体制造良率预测与关键特征挖掘 —— 项目计划书

---

## 一、项目背景

### 1.1 业务背景

在晶圆制造流程中，从前端光刻、刻蚀、沉积到后端封装测试，每一道工序都会由数百个传感器实时记录温度、电压、电流、气体流量、压力、膜厚等工艺参数。晶圆在完成整线制造后进入晶圆测试及成品测试环节，由测试设备给出"通过（Pass）"或"失效（Fail）"的结果。
由于半导体制造的高良率特征，失效样本极低，呈现典型的 **极端不平衡二分类问题**。尽早识别出潜在失效晶圆，能够：

- 降低不必要的后续封装与测试成本；
- 及时追溯造成失效的工艺环节，为工程改善提供依据；
- 建立工艺参数与良率之间的量化模型；
- 逐步实现"虚拟量测"与"预测型维护"。

本项目使用的 **SECOM** 数据集是公开的半导体制造传感器数据基准数据集，广泛用于学术研究与工业项目验证。

### 1.2 数据概况

| 项目 | 内容 |
|------|------|
| 数据来源 | SECOM 公开数据集（UCI Machine Learning Repository） |
| 样本数 | 1567（=1511 有效样本 + 56 被移除异常样本） |
| 原始传感器特征数 | 590 列 |
| 标签列 | 1 列（−1=良品 / 1=失效） |
| 良品样本 | 约 1463 个 |
| 失效样本 | 约 104 个 |
| 不平衡比 | **约 14.1 : 1** |
| 稀疏性 | 平均缺失率 4.54%，最大缺失率 91.19%，52 列无缺失 |
| 特征类型 | 全部为浮点数值，多数为工艺监测信号 |
| 数据格式 | 空格分隔文本文件（secom.data / secom_labels.data） |

**EDA 结论预览**：

1. 样本严重不平衡，常规分类器会倾向于把全部样本预测为"良品"；
2. 数据有大量缺失且存在异常值（部分传感器列缺失率超过 70%）；
3. 特征之间存在较强相关性，部分列相互冗余；
4. 特征尺度差异巨大（均值范围从 −5667 到 3014 不等），必须进行标准化。

## 二、项目目标

### 2.1 核心目标

| 目标 | 指标 | 目标值 | 说明 |
|------|------|--------|------|
| **识别失效能力** | **Recall（召回率 / 查全率）** | **≥ 90%** | 在测试集中，真正的失效晶圆被模型识别出来的比例，代表"不漏检"能力 |
| **综合分类质量** | **AUC-ROC** | **≥ 0.80** | 模型整体的区分度 |
| **综合稳定性** | **G-Mean** | **≥ 0.70** | √(Recall × Specificity)，衡量两类平衡表现 |

### 2.2 附加目标

- 给出 **Top-20 关键失效特征**，供工艺工程师参考；
- 给出 **SHAP 可解释性分析**，解释每个预测的驱动因子；
- 给出 **完整可视化**（ROC/PR 曲线、校准曲线、混淆矩阵）；
- 提供 **工程建议**（哪些工序/传感器需要重点监控）。

### 2.3 业务约束

- **Recall 优先**：漏检代价远高于误检代价，优先保证recall，因此 Recall 必须 ≥ 90%；
- **不可过拟合**：测试集只在最终评估环节使用一次，所有模型选择、阈值、校准均基于训练集 OOF（Out-of-Fold）概率；
- **严格避免信息泄露**：划分后再做特征选择，不使用测试集统计量。

## 三、总体技术路线

```
原始数据 (secom.data)
   │
   ▼
阶段 1：EDA 可视化 + 预处理
   ├─ 标签分布分析（饼图/柱状图）
   ├─ 缺失率分析（直方图 + 累积分布）
   ├─ 样本缺失分布（按标签分组 + 时间顺序）
   ├─ 缺失率过滤（丢弃缺失率 > 70% 的列）
   ├─ KNN 填补（k=5，加权欧氏距离）
   ├─ 异常样本过滤（每行超过 |z|>5 的特征数 > 8 的样本移除）
   ├─ 低方差过滤（Variance Threshold < 1e-8）
   ├─ 高相关过滤（Pearson |r| > 0.97 的冗余列移除）
   └─ Z-Score 标准化（均值 0，方差 1）
   │
   ▼
阶段 2：数据划分 + 组合特征选择
   ├─ Stratified 划分（80% 训练 / 20% 测试，保持正负比例）
   └─ Borda Count 组合特征选择（5 种方法投票，取 Top-120）
       ├─ 方法 1：互信息 MI（捕捉非线性相关）
       ├─ 方法 2：ANOVA F-value（线性相关性）
       ├─ 方法 3：Random Forest Gini Importance
       ├─ 方法 4：XGBoost 分裂增益
       └─ 方法 5：Extra Trees
   │
   ▼
阶段 3：不平衡采样策略 + 模型配置
   ├─ 4 种采样（None / SMOTE / ADASYN / Borderline-SMOTE）
   └─ 5 种模型
       ├─ XGBoost (max_depth=3, n_estimators=600, scale_pos_weight)
       ├─ XGBoost (max_depth=4, n_estimators=500)
       ├─ Extra Trees (max_depth=12, class_weight='balanced_subsample')
       ├─ SVM-RBF（C=1.0，class_weight='balanced'）
       └─ Logistic Regression（C=0.3，class_weight='balanced'）
   │
   ▼
阶段 4：多配置网格搜索（5-Fold CV × 10 seeds）
   ├─ 对 4 采样 × 5 模型 = 20 配置
   ├─ 每种配置 × 10 随机种子，进行 5-Fold Stratified CV
   ├─ 得到 OOF 概率（10 seed 平均）
   └─ 在完整训练集重训练 → 预测测试集概率（10 seed 平均）
   │
   ▼
阶段 5：Isotonic 概率校准 + 多目标阈值优化
   ├─ 对每个配置的 OOF 概率进行 Isotonic 保序回归校准
   ├─ 约束 Recall ≥ 0.90，最大化综合得分：
   │    Score = 0.25 × Precision + 0.45 × G-Mean + 0.30 × F1
   └─ 输出每个配置的最佳阈值和多维指标
   │
   ▼
阶段 6：Stacking 集成 vs 简单平均集成
   ├─ 取 Top-12 配置的概率作为元特征（形状 N × 12）
   ├─ Stacking：L1-正则化 LR（搜索 C ∈ {0.01, 0.05, 0.1, 0.5, 1.0, 5.0}）
   ├─ 简单平均：Top-12 配置概率算术平均
   ├─ 元分类器输出再做一次 Isotonic 校准
   └─ 选择 G-Mean 更高的策略作为最终方案
   │
   ▼
阶段 7：测试集最终评估（仅一次）
   ├─ 使用训练集 OOF 上获得的最佳阈值预测测试集
   ├─ 输出完整报告：Recall/Precision/F1/G-Mean/AUC-ROC/AUC-PR/
   │               Specificity/MCC/Balanced Acc/Brier Loss
   └─ 可视化：ROC/PR 曲线、混淆矩阵热力图、校准曲线
   │
   ▼
阶段 8：SHAP 可解释性分析
   ├─ 选取最佳基于树的模型（XGBoost d3 / Borderline-SMOTE）
   ├─ 在完整训练集（+采样）上重新训练用于解释
   ├─ SHAP TreeExplainer 计算测试集 SHAP 值
   ├─ 输出 Top-20 SHAP 重要性（全局关键特征）
   └─ SHAP Beeswarm 图（每个特征的正负贡献分布）
```

## 四、阶段详细方案

### 阶段 1：EDA 可视化与预处理

| 步骤 | 方法 | 参数 | 目的 |
|------|------|------|------|
| 1.1 标签分布 | 饼图 + 柱状图 | — | 直观展示不平衡比 |
| 1.2 缺失率分析 | 每列缺失率直方图 + 累积分布 | 阈值 70% | 识别大量缺失的传感器列 |
| 1.3 样本缺失分析 | 按样本的缺失散点图 + 箱线图（Pass/Fail 分组） | — | 判断缺失值是否与标签相关 |
| 1.4 缺失率过滤 | 删除缺失率 > 70% 的列 | 阈值 70% | 丢弃高缺失列 |
| 1.5 KNN 填补 | KNNImputer（欧氏距离 + 距离加权） | **k=5** | 填补剩余 NaN，避免均值填补抹平方差 |
| 1.6 异常样本过滤 | 计算每行 |z|>5 的特征数量，>8 的整行移除 | 阈值 **>8** | 剔除采集异常的晶圆记录 |
| 1.7 低方差过滤 | VarianceThreshold | **< 1e-8** | 移除常量/近常量传感器列 |
| 1.8 高相关过滤 | 计算 Pearson 相关矩阵，移除 |r|>0.97 的冗余列 | 阈值 **0.97** | 减少多重共线性 |
| 1.9 标准化 | StandardScaler（μ=0，σ=1） | — | 统一尺度以便后续 KNN/SVM/LR 正常工作 |

**预期数据形状变化**：

| 阶段 | 样本数 | 特征数 |
|------|--------|--------|
| 原始 | 1567 | 590 |
| 缺失率过滤后 | 1567 | ≈ 582 |
| KNN 填补 + 异常样本过滤后 | ≈ 1515 | ≈ 582 |
| 低方差过滤后 | ≈ 1515 | ≈ 460 |
| 高相关过滤后 | ≈ 1515 | ≈ 314 |
| **最终（进入建模）** | **≈ 1515** | **≈ 314** |

### 阶段 2：数据划分与特征选择

- **划分方式**：Stratified 80/20（训练集 ~1212 样本，测试集 ~303 样本，其中正例约 19 个）；
- **特征选择必须在划分后进行**（严格避免信息泄露）；
- **组合特征选择（Borda Count 投票）**：对 5 种方法各自的特征排序求"排位和"，取 Top-120；

### 阶段 3–4：多模型 × 多采样网格搜索

- **模型池**：XGBoost d3、XGBoost d4、Extra Trees deep、SVM-RBF、LR（共 5 种）
- **采样池**：不采样、SMOTE、ADASYN、Borderline-SMOTE（共 4 种）
- **共 20 种配置**，每种配置进行 5-Fold CV × 10 随机种子，得到 OOF 与测试集概率
- **关键参数**：
  - XGBoost scale_pos_weight = 负样本数 / 正样本数
  - Extra Trees：class_weight='balanced_subsample'
  - SVM：class_weight='balanced'，max_iter=5000
  - LR：class_weight='balanced'，L2 正则，saga 求解

### 阶段 5：概率校准与阈值优化

- **校准方法**：Isotonic Regression（保序回归），基于 OOF 概率拟合，再校准 OOF 与测试概率；
- **阈值搜索**：遍历候选阈值，在满足 **Recall ≥ 0.90** 约束下最大化综合得分：

  **Score = 0.25 × Precision + 0.45 × G-Mean + 0.30 × F1**

- 若无满足约束的阈值，则回退为最大化 G-Mean。

### 阶段 6：集成策略

将 Top-12 配置的概率作为元特征（维度 1212 × 12）：

| 策略 | 方法 | 说明 |
|------|------|------|
| **Stacking** | L1-正则化 LR 元分类器，搜索 C | 学习各配置的权重，对相似配置自动降权 |
| **简单平均** | Top-12 概率算术平均 | 鲁棒 baseline |
| **再校准** | 元分类器输出再次 Isotonic 校准 | 防止集成后概率偏离 |
| **选择规则** | 取 G-Mean 更高的策略 | 符合业务平衡指标 |

### 阶段 7：测试集评估

在最终方案（Stacking 或 简单平均）的测试集概率上，使用训练集 OOF 上确定的最佳阈值进行硬分类，输出：

- 混淆矩阵：TN / FP / FN / TP
- 核心指标：Recall、Precision、G-Mean、F1、Specificity、AUC-ROC、AUC-PR、MCC、Balanced Accuracy、Brier Loss
- 可视化：ROC 曲线、PR 曲线、混淆矩阵热力图、校准曲线

### 阶段 8：SHAP 可解释性与关键特征挖掘

- 使用 **最佳基于树的配置**（XGBoost d3 / Borderline-SMOTE）在完整训练集（+采样）上重新训练；
- 使用 **SHAP TreeExplainer** 计算测试集所有样本的 SHAP 值；
- 输出 Top-20 全局 SHAP 重要性（关键失效特征）；
- 绘制 SHAP Beeswarm 图，展示每个特征取值高低对"失效"预测概率的贡献方向与强度。

## 五、评价指标体系

| 指标 | 公式/含义 | 关注点 |
|------|-----------|--------|
| **Recall（查全率）** | TP / (TP + FN) | **核心业务指标**，代表失效晶圆的检出率 |
| **Precision（查准率）** | TP / (TP + FP) | 报警的"可信程度"，高 Precision 意味着误检少 |
| **F1-Score** | 2 × (P × R) / (P + R) | P/R 调和平均，综合指标 |
| **G-Mean** | √(Recall × Specificity) | 不平衡数据的重要综合指标 |
| **AUC-ROC** | ROC 曲线下面积 | 整体区分度 |
| **AUC-PR** | PR 曲线下面积 | 不平衡场景下更严苛的区分度 |
| **Specificity** | TN / (TN + FP) | 良品被正确判断的比例 |
| **MCC** | Matthews 相关系数 | −1 到 +1，对不平衡鲁棒 |
| **Balanced Accuracy** | (Recall + Specificity) / 2 | 对不平衡更公平的准确率 |
| **Brier Loss** | MSE(预测概率, 真实标签) | 概率校准质量 |

**各阶段侧重点**：

- 单一模型评估：重点关注 AUC-ROC、G-Mean
- 阈值/校准优化：重点满足 Recall ≥ 0.90
- 最终测试：完整输出全部指标

## 七、风险与对策

| 风险 | 描述 | 影响 | 对策 |
|------|------|------|------|
| **R1 极端不平衡** | 正例仅 ~104 个，训练不足 + 测试波动大 | Recall/PR 指标方差大 | 多采样策略 + 多模型 + 10 seeds 平均；Stacking 集成 |
| **R2 信息泄露** | 在划分前做特征选择/标准化会泄露测试信息 | 泛化能力虚高 | 严格"先划分再处理"，所有统计量仅拟合训练集 |
| **R3 阈值敏感** | 不同决策阈值下 Recall/Precision 此消彼长 | 人工选择存在主观 | 约束优化 + 多目标得分，搜索 0.001–0.999 细粒度阈值 |
| **R4 传感器噪声** | 采集异常、漂移导致极端值 | 模型鲁棒性降低 | 每行 z-score 过滤 + KNN 填补，避免移除正例样本 |
| **R5 高维多重共线性** | 相关传感器信号高度冗余 | 树模型影响小，但 SVM/LR 不稳定 | Pearson 阈值 0.97 去冗余，L1 正则元分类器进一步抵抗 |
| **R6 过拟合** | 多次搜索超参数可能泄露 OOF 本身 | 真实泛化下降 | 最终测试集仅用一次；保留简单平均集成作为鲁棒 baseline |

## 十一、结论

本项目围绕 SECOM 数据集，构建了一条从数据清洗 → 组合特征选择 → 多模型 × 多采样搜索 → Isotonic 校准与阈值优化 → Stacking 集成 → SHAP 可解释性分析的完整机器学习流水线。核心目标是在 **Recall ≥ 90% 的硬性约束下**，最大化综合分类表现（AUC-ROC ≥ 0.80，G-Mean ≥ 0.70），并为工艺工程师提供可解释的关键失效特征列表。

通过严格的训练-测试隔离、10 随机种子平均、多模型集成与保序校准，本项目力求在不平衡、高维、噪声的半导体制造场景下，输出稳健的良率预测模型。
